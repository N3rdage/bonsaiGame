Two rules, both learned from corrections.

**Rule 1 — only at end of session, not end of task.** The rhyme is a
*farewell*. Don't drop it after a merged PR or a finished task when more
work is queued in the same session — it reads as "trying to get rid of"
Drew. Save it for genuine sign-offs (Drew is wrapping up for the day,
explicitly saying goodbye, etc.).

**Rule 2 — call-and-response order.**
- **Initiator:** "See you later, alligator."
- **Response:** "In a while, crocodile."

If I'm saying goodbye first, use the alligator line. If Drew has already
said the alligator line, respond with the crocodile line. The joke is
in the call-and-response — using the response as the opener reads as off.

**Why:**
- Rule 1: Drew flagged that I dropped "See you later, alligator" right
  after a merged PR while we were obviously about to start the next one.
  Made the hand-off feel dismissive (he name-checked the Terminator
  movies — the bot trying to wrap things up unprompted).
- Rule 2: Drew called this out after I closed an earlier session with
  "In a while, crocodile" when I should have used the alligator line
  since I was the one signing off.

**How to apply:**
- Don't initiate the rhyme between PRs or tasks. End of a PR is "ready
  for the next one", not "goodbye".
- Use it only when the session is ending — Drew signals "wrapping up",
  "done for today", or otherwise clearly closes things.
- When Drew initiates with the alligator line, reply with the crocodile
  line. When I'm initiating a genuine farewell, use the alligator line.
